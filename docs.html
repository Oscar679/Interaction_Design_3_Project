Abstract Service Class

Base class that the other services extend. Can't be instantiated on its own — if you try, it throws an error. Any subclass needs to implement fetchData() or it'll throw as well.

SheetService Class

Fetches sensor data from a published Google Spreadsheet. The response comes back as JSONP, so it strips the wrapper and parses out the temperature and humidity rows.

SMHIService Class

Fetches weather data from the SMHI Open Data API. Takes a parameter (temperature or humidity), station ID, and time period. Used to pull official data to compare against the campus sensor.

SensorChart Class

Web Component that renders the campus sensor chart with ApexCharts. Subscribes to the Observable so it updates when the user switches between Day/Month. The selected period is saved in localStorage and restored on load.

SMHIChart Class

Same idea as SensorChart but for the SMHI data. Uses a periodMap to translate "Day"/"Month" into the API's period strings (latest-day, latest-months). Also subscribes to the Observable for period changes.

Observable Class

Central piece that ties the charts together. Only one instance exists (Singleton), and charts subscribe to it. When the user picks a new period or hits refresh, it loops through all subscribers and tells them to update.
